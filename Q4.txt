
—¿Cómo elimino una rama creada en git? 
    Crea una nueva rama llamada feature/QA, para posteriormente borrarla.

git branch feature/QA
git branch -d feature/QA

—¿Por qué debería de usar la ramificación en proyectos?

Las ramas permiten trabajar en nuevas funcionalidades o correcciones sin afectar el código principal
facilitando la colaboración y el control de versiones.

—Usted recibió un ticket [SEM-005] y una vez terminado el trabajo usted decide crear una rama 
para un posterior PR. ¿Qué nombre pondría a la rama al momento de crearla?
